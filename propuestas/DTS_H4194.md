# #DTS #H4194 App web para agentes comerciales

Los objetivos de este proyecto son:
+ Disponer de una aplicación que sirva a los comerciales como herramienta para registrar y llevar el seguimiento de sus ventas.

## Propuesta realizada a petición de:
Roberto y Aritz

Te paso mis notas hasta ahora fruto de los últimos correos que intercambiamos:

+ Clientes: Cada comercial ve sus clientes. Podemos ver si es interesante acceder a datos del cliente o documentos (pedidos, presupuestos, tratos, etc) desde su ficha
    + Listado de últimas compras (líneas de factura) con datos similares a los que se ven en el ERP > Factracuón > Más > Histórico de ventas (últimas ventas para ver qué compra y a qué precio)

+ Clientes: Modificar precios de artículos por cliente. Modificable según perfil

+ (NO PRIORITARIO) Presupuestos: Cada comercial puede ver sus presupuestos, hacer nuevos y aprobarlos (pasar a pedido)

+ (VERLOS y VER SU ESTADO) Pedidos: Cada comercial puede ver sus pedidos, hacer nuevos y marcarlos como enviado a central. En central aparecerán como pendientes de revisión o como pedidos definitivos `(por aclarar)`. Funcionamiento:
    + Borrador: El pedido lo está preparando el comercial, puede modificarlo o borrarlo, y en central no se puede servir (digamos que el agente lo está preparando pero no ha terminado de hacerlo).
    
    + Pte aprobación: El agente ha terminado de hacer el pedido y lo manda a la central. Ahí un responsable revisa el pedido (generalmente es por si ha puesto descuentos demasiado altos o cosas así). El responsable puede eliminar el pedido o pasarlo a Aprobado. El pedido tampoco se puede servir en este estado. El agente no lo puede modificar tampoco en este estado.
    
    + Aprobado: El responsable ha aprobado el pedido y este ya se puede servir.

+ Selección de elementos del pedido en función del histórico de compras de cliente

+ NO NECESARIO: Figura de agente supervisor. Cada agente puede tener otro agente que actúa como supervidor, un agente supervisor ve sus datos y los de todos sus supervisados.

+ OPCION DESCARGA: Albaranes y facturas de cliente: Solo lectura, generar PDF (no se puede adjuntar automáticamente a whatsap o correo).

+ NO: Albaranes y facturas de cliente: Enviar enlace para descargar. Publicar los documentos para su descarga en enlaces aleatorios, y enviar solo en enlace, como hace Telefónica, Iberdrola, etc. Sería copiar al portapapeles un enlace tipo https://www.dtsoabe.com/facturas/SSD458ASDASD y en el correo / mensaje indicar "Puedes descargar la factura en " y pegar el enlace. El cliente podrá descargarlo y podemos asignar una caducidad a estos enlaces.
 
 
+ COMPARATIVA de MES por agente, cliente (como el del ERP). Informes para los comerciales: Serán informes fijos, no se podrá cambiar ningún parámetro. Informe comparativo de ventas de este mes, hasta hoy desde el 01/01/de este año y de este año.
 

+ Tratos u oportunidades de ventas. Son posibles ventas alrededor de las cuales se introduce información.

    + Comentarios: Trato u Oportunidad de venta, que al final es una venta que puede o no hacerse, o incluso puede ser una incidencia o problema del cliente que no se va a cobrar pero que puede tener tareas asociadas. El trato puede estar asociado a un cliente o cliente potencial, y si se completa puede genera un presupuesto o pedido.

+ Tareas de tratos: Son tareas de llamadas, correos, reuniones asociadas a tratos. Las tareas se pueden integrar con Google Calendar.

    + Comentarios: Asociados al trato, puedes crear tareas, que al final son comunicaciones por distintos canales. Lo bueno de asociarlas al trato es que en la ficha del trato el agente ve el timeline de tareas y comunicaciones de esa operación. Las tareas tienen un estado Pendiente/Hecha y una fecha de ejecución. Cualquier trato "vivo" debe tener al menos una tarea Pendiente (aunque sea revisar si el cliente ha respondido).

    Hacer una parte de tareas pendientes (acabamos de estar con un cliente y nos dice que le pasemos precio para un productos especial o una pieza que no tenemos y que hemos de buscar, o ...). Abrir una tarea con un recordatorio.
 

 + Home de página para comerciales, con:

    + Tareas y tratos abiertos
    + Informes resumen de cómo van las ventas respecto a previsión (si la hay)

 
HERRAMIENTAS DE CONTROL DE LAS VENTAS Y LOS COMERCIALES PARA EL DIRECTOR COMERCIAL:
 
Básicamente es un sistema de objetivos anuales que puede hacerse tan complejo como sea necesario. Los objetivos podrían ser mensuales / trimestrales / etc. Se podrían poner objetivos por familias de productos concretas, etc. Luego habría que ver cómo plasmamos el grado de progreso en la aplicación, para que el agente sepa cómo va, y en los informes de seguimiento para el agente comercial que queráis realizar. Siempre son comparativas de la facturación real con la esperada según los objetivos establecidos.
 
Os puedo enseñar de el ejemplo que vimos ayer, la parte de backoffice de definición de objetivos.
También tenemos alguna excel de KPIs (tratos ganados respecto a perdidos, etc) para seguimiento de los agentes.
