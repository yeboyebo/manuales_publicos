# #DTS #H4194 App web para agentes comerciales

Los objetivos de este proyecto son:
+ Disponer de una aplicación que sirva a los comerciales como herramienta para registrar y llevar el seguimiento de sus ventas.

## Propuesta realizada a petición de:
Roberto y Aritz

## Estructura

### Webapp / Home
La Home dará acceso a las siguientes opciones:

+ Clientes
+ Presupuestos
+ Pedidos
+ Informe de comparativa de ventas
+ Artículos


### Webapp / Clientes
Se mostrará un listado / detalle de clientes

Filtro de acceso: Por agente del usuario o todos si el agente es superusuario.

El detalle de cliente mostrará:
    
    + Pestaña Principal
    + Pestaña Comercial
    + Pestaña Direcciones
    + Pestaña Cuentas bancarias
    
    + Últimas compras:
        + Referencia + descripción
        + Cantidad x precio unitario = Total (+ descuentos)
        + Fecha factura

        Ordenadas por fecha descendente
    
    + Precios por cliente:
        Referencia + descripción
        Precio (editable)

    + Pestaña Documentos / Albaranes del cliente:
        + Código
        + Importe total
        + Fecha
        + Botón Descargar

    + Pestaña Documentos / Facturas del cliente:
        + Código
        + Importe total
        + Fecha
        + Botón Descargar

Más casos de uso generales:

+ Crear cliente.
    * Params:
        * Nombre
        * Tipo Id. Fiscal
        * Id. Fiscal
        * Dirección (opcional)
    * Validaciones:
        * 
    * Acciones:
        * Retorna IdCliente

+ Cambiar nombre del cliente
    * Params:
        + IdCliente
        + IdGrupo
    + Validaciones:
        + El cliente existe

+ Cambiar Id.Fiscal del cliente
    * Params:
        + IdCliente
        + Tipo Id. Fiscal
        + Id. Fiscal
    + Validaciones:
        + El cliente existe

+ Cambiar observaciones cliente
    * Params:
        + IdCliente
        + IdGrupo
    + Validaciones:
        + El cliente y el grupo existen

+ Cambiar grupo cliente
    * Params:
        + IdCliente
        + Observaciones
    + Validaciones:
        + El cliente existe

+ Dar de baja cliente
    * Params
        + IdCliente
        + Fecha de baja
    + Validaciones:
        + El cliente existe y está activo

+ Reactivar cliente
    * Params
        + IdCliente
    + Validaciones:
        + El cliente existe y está de baja

+ Asociar contacto principal
    + Params:
        + IdCliente
        + IdContacto
    + Validaciones:
        + Cliente y contacto existen
    + Acciones:
        + El contacto se asocia al cliente si no lo está
        + El contacto pasa a ser contacto principal
         
+ Cambios genéricos por clave de entidad asociada:
    + Forma de pago
    + Divisa
    + Serie
    + Grupo IVA Negocio

+ Cambiar riesgo máximo
    + Riesgo
+ Cambiar agente comercial
    + Agente comercial


### Webapp / Presupuestos
Cada comercial puede ver sus presupuestos, crearlos y aprobarlos.

Casos de uso de mantenimiento y aprobación de presupuestos.

### Webapp / Pedidos
Cada comercial puede ver sus pedidos, y el estado (Servido / Pendiente / Parcial) en el que están a nivel de línea.

No podrá editarlos ni crear nuevos pedidos.

### Webapp / Informe comparativa de mes
Misma opción que el informe "Comparativa de ventas" del ERP restringida al agente conectado.

### Webapp / Artículos
Se mostrará un listado / detalle de artículos

Filtros: Referencia, descripción

El detalle de artículo mostrará:

    + Referencia
    + Descripción
    + Familia
    + Precio de tarifa
    + IVA
    + Precios por cliente
