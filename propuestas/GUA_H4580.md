# #GUA #H4580 Sistema de tipos de atributo por familia y sincronización con Magento

Los objetivos de este proyecto son:
+ Disponer de un sistema de atributos asociados a artículos que permita su filtrado en la tienda online

## Propuesta realizada a petición de:
Almudena

## Estructura

### Almacén / Familias y Atributos
Crearemos las siguientes tablas asociadas a cada familia:

Familias:
    + Id Magento

Tipos de atributo
    + Id Tipo de atributo
    + Nombre
    + Id Magento

Tipos de atributo por familia
    + Id Familia
    + Id Tipo de atributo

Valores por tipo de atributo
    + Id Tipo de atributo
    + Id Magento
    + Id tipo de atributo
    + Valor

Valores de atributo por artículo
    + Referencia
    + Id Tipo de atributo
    + Id Valor de atributo

### Almacén / Artículos
Crearemos un apartado de atributos en el que el usuario podrá escoger para cada tipo de atributo asociado a su familia.

### Almacén / Cambios masivos
Crearemos una opción que permitirá seleccionar una familia y establecer sus valores de atributos, y aplicarlo a los artículos que cumplan el filtro.

### ERP / Sincro con Magento
Los artículos serán encolados para sincronizar sus atributos cuando estos se cambien en su ficha o a partir de cambios masivos.

## Asistencia a puesta en marcha
Se realizará un seguimiento de la primera carga de datos para asegurarnos de que los artículos pasan de simples a compuestos de forma correcta, y conservando el resto de campos en Magento.

## Puntos destacados 
Es importante indicar que los datos del catálogo de tipos de atributos y valores por tipo de atributo NO se sincronizarán con Magento. Yeboyebo los creará en Magento con la carga inicial y cuando se cree un nuevo tipo será necesario avisar a Yeboyebo para crear el equivalente valor en Magento.
