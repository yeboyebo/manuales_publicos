# #PRE #H4744 Importación de facturas generadas desde Vitaly

Los objetivos de este proyecto son:

+ Permitir importar facturas creadas desde Vitaly en las bases de datos de Prevecam y Prevecam Formación

## Propuesta realizada a petición de:
Fran

## Estructura

### Facturas de venta / Infraestructura
+ Modificaremos la longitud de los campos codigo de factura y tablas asociadas a 13 caracteres
+ Modificaremos la longitud del campo numero de factura 13 caracteres
+ Craeremos una serie _IF - Importación Facturas_

### Facturas de venta / Importación
Añadiremos a la pantalla los siguientes controles:

+ Botón Importar

El botón permitirá importar un fichero CSV o pegar unas columnas de excel con el siguiente formato:

    * FC_NUMERO_DOCUMENTO: Código de factura
    * FC_ENTIDAD. Se ignorará.
    * FC_CLIENTE_CIF: CIF del cliente. Se comprobará que existe en clientes y está asociado a un único cliente
    * FC_CLIENTE_NOMBRE.
    * FC_CLIENTE_DIRECCION_SOCIAL
    * FC_CLIENTE_PROVINCIA_SOCIAL
    * FC_FECHA: Fecha de la factura
    * FC_FECHA_VCTO. Se ignorará
    * FC_CUANTIA_TOTAL: Se usará para comprobar la factura tras la importación
    * FC_CUANTIA_TOTAL_IMPUESTOS: Se usará para comprobar la factura tras la importación
    * FL_DESCRIPCION: Descripción de la línea.
    * FL_CUENTA_CONTABLE. Se ignorará.
    * FL_LINEA_NEGOCIO. Se ignorará.
    * FL_LINEA_NEGOCIO_NOMBRE. Se ignorará.
    * FL_PORCENTAJE_IMPUESTO. Se usará para buscar el código de impuesto y el IVA de la línea.
    * FL_CUANTIA_TOTAL. Se usará como precio unitario de la línea (cantidad siempre es 1)
    * FL_CUANTIA_TOTAL_IMPUESTOS. Se ignorará.

Proceso de importación:

    + Comprobaciones por línea
        + Comprobación de que los datos no ignorados están informados
        + Comprobación de cliente encontrado por CIF
        + Comprobación de que el FL_PORCENTAJE_IMPUESTO existe en la tabla de impuestos
        + Comprobación de que FC_NUMERO_DOCUMENTO no existe ya en la tabla de facturas
        + Comprobación de que FC_FECHA está comprendida en el ejercicio activo de Eneboo
    + Preproceso:
        + Ordenación de las líneas por FC_NUMERO_DOCUMENTO
    + Comprobaciones globales: Agrupando por FC_NUMERO_DOCUMENTO, la suma de FL_CUANTIA_TOTAL de las líneas coincide con FC_CUANTIA_TOTAL
    + Proceso:
        + Para cada factura, crear cabecera con:
            + Cliente obtenido de búsqueda por CIF. Datos de cliente (serie, forma pago, dirección, CIF) obtenidos de su ficha
            + Fecha con la FC_FECHA
            + Serie IF
            + Ejercicio. Ejercicio activo de Eneboo.
            + Número: Código copleto de FC_NUMERO_DOCUMENTO
            + Líneas con:
                + Referencia nula
                + Descripción con FL_DESCRIPCION
                + Código de impuesto obtenido de la tabla de impuestos según FL_PORCENTAJE_IMPUESTO
                + IVA: FL_PORCENTAJE_IMPUESTO
                + Recargo: 0
                + Cantidad: 1
                + Precio unitario: FL_CUANTIA_TOTAL
                + Descuentos: 0
        + Postproceso por factura:
            + Comprobar que el neto coincide con FC_CUANTIA_TOTAL
            + Comprobar que el total con IVA coincide con FC_CUANTIA_TOTAL_IMPUESTOS
