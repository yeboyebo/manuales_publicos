# #SAN #H4599 Motivos de devolución

Los objetivos de este proyecto son:

+ Que los comerciales puedan indicar cuál es el motivo de cada devolución que registran, de forma que este dato se registr para su posterior análisis.

## Propuesta realizada a petición de:
Paloma

## Estructura

### Webapp / Motivos de devolución
Crearemos una nueva opción desde la que gestionar lo motivos de devolución. Los campos de esta tabla son:

    + (id interno)
    + Tipo de motivo: Interno / Externo. Interno cuando la responsabilidad es de Sanhigia, Externo cuando es del cliente
    + Descripción del motivo
    + Otros (Sí / No). Indica que se trata de la opción Otros. Solo puede haber un registro con Otros = Sí por cada tipo de motivo.

### Facturación / Pedidos
Añadiremos a la tabla los siguientes campos:

+ Motivo de devolución. Relacionado con Motivos de devolución.
+ Descripción del motivo de devolución (para el caso otros)

### Webapp / Devoluciones
Cuando el usuario pulsa el botón de Confirmar devolución se abrirá un diálogo para especificar el motivo de la devolución.

El formulario estará dividido en dos secciones, que mostrarán respectivamente los posibles motivos internas y externas.

El usuario deberá escoger un motivo interno o externo. Si elige una de las dos opciones Otros, deberá indicar la descripción del motivo en un control de texto asociado.

Los datos de la opción elegida y de la descripción del motivo (casos Otros) se guardarán en el pedido.
