# #SAN #H4623 Volcado diario de resultados de previsión de compras a tabla

Los objetivos de este proyecto son:

+ Disponer de un proceso diario que calcule las necesidades de productos y vuelque la cantidad a pedir y resto de propiedades a la tabla necesidades_stock

## Propuesta realizada a petición de:
Jesús

## Estructura

### Tabla de necesidades

Correspondencias entre columnas y campos:

+ Referencia: referencia
+ A pedir: necesidad
+ T1: t1 (nuevos campos t1 - t4)
+ T2: t2
+ T3: t3
+ T4: t4
+ Actual: unidadesalmacenadas
+ Por servir: reservada
+ Por recibir: pterecibir
+ Días: diasdeservicio

### Proceso de carga de la tabla
Crearemos un proceso de carga que realizará las mismas consultas y ajustes que la actual pantalla de previsión de compras, volcando los resultados a la tabla de necesidades stock

### Log de ejecución
Se incluirá un log de ejecución que se guardará tras cada ejecución para comprobar si el script se ejecuta

### Fecha de última carga
Se incluirá un nuevo campo en Facturacion > Almacén > Configuración:

+ Última carga de tabla de necesidades de stock

El proceso actualizará este valor tras cada ejecución con éxito.

### Instalación
Se instalará y configurará para que se lance diariamente el proceso de carga (primero sobre la BD de pruebas y luego sobre producción).
