# #SAN #H4641 Funcionalidad de pérdida de presupuesto y mostrar presupuestos, tratos y tareas pendientes en clientes

Los objetivos de este proyecto son:

+ Disponer en la opción de clientes de la webapp de los datos asociados al cliente en cuanto a:
    + Tratos pendientes
    + Tareas pendientes
    + Presupuestos pendientes

+ Calificar los presupuestos perdidos como tales, de forma que no sigan apareciendo como pendientes.

## Estructura

### ERP. Presupuestos perdidos
Crearemos un nuevo campo en presupuestos de cliente:

+ Estado. Posibles valores
    + Pendiente (valor por defecto)
    + Aprobado
    + Perdido

Añadiremos a la pantalla de maestro de presupuestos un botón "Perdido/Reactivar".

+ Habilitaciones: El botón estará habilitado cuando el estado del presupuesto es Perdido o Pendiente.

+ Etiqueta del botón: Según el estado del presupuesto, la etiqueta será:
    + Perdido, cuando el estado es Pendiente
    + Reactivar, cuando el estado es Perdido

+ Acciones: Al pulsar el botón:
    + El presupuesto cambia de estado:
        + A Perdido, si el estado previo es Pendiente
        + A Pendiente, si el estado previo es Perdido.

+ Campo. Descripción.

Migración inicial: ver updates en notas de desarrollo

### CRM. Tratos, tareas y presupuestos en ficha de cliente
Incluiremos tres subtablas en las fichas de clientes que permitirán:

+ Ver los tratos pendientes asociados.
+ Ver las tareas pendientes asociados.
+ Ver los presupuestos pendientes asociados.

```sql
update presupuestoscli set estado='Aprobado' where not editable;
update presupuestoscli set estado='Perdido' where editable and fecha < 'X';
```
