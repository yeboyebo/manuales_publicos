# #SAN #H4657 Inclusión de licencias de farma en el flujo de creación (Make)

Los objetivos de este proyecto son:

+ Incluir al ERP en el actual flujo de trabajo para la creación de licencias de farma.
+ Controlar desde el ERP el estado del proceso de creación de cada licencia

## Propuesta realizada a petición de:
Paloma

## Estructura

### Licencias de farma / Tabla y cálculo de estado
Actualmente las licencias de farma solo son visibles desde las fichas de Cliente. Las modificaremos para que puedan ser consultadas de forma independiente desde una nueva opción _Licencias_ en Facturación / Principal.

Nuevos campos

+ Cod. Agente. Relacionado con Agentes
+ Nombre cliente.
+ Cod. Cliente (modificado para permitir licencias sin cliente, en estado preliminar)
+ Fecha de datos clínica revisados y envío de acuerdos
+ Fecha de recepción de acuerdos
+ Fecha de envío de documentación para presentar
+ Estado. Calculado como:
    + _En revisión_ (Fecha de datos clínica revisados no informada)
    + _Acuerdo por recibir_ (Fecha de recepción de acuerdos no informada)
    + _Por presentar_ (Fecha de envío de documentación para presentar no informada)
    + _Presentada_ (todas las fechas informadas)

### Licencias de farma / Integración de creación de licencia
Se configurará una función de API que permitirá a un nodo _Make_ generar la licencia y su trato asociado.

Parámetos de entrada:

    + Nombre del cliente.
    + Id del agente asociado.

Parámetros de salida:

    + Id de la licencia creada


### Licencias de farma / Opción Licencias en Webapp
Crearemos una opción de menú para ver las licencias de farma.

Cada agente podrá ver solo sus licencias.

El grupo ¿? podrá ver todas las licencias

Los agentes podrán:

    + Estado _En revisión_:
        + Pulsar un botón Datos revisados
            + Solo podrán hacer esto cuando la licencia esté asociada a un cliente ¿y el trato a un contacto?

    + Estado (cualquiera):
        + Consultar el estado de la licencia y acceder desde ahí a su trato.

Los usuarios del grupo (X) podrán:
    + Marcar las fechas de recepción y envío de documentación.

El formulario de licencia mostrará todos sus campos y un resumen de su trato asociado, si lo tiene, con un botón de ir al trato para editarlo.


### Licencias de farma / Integración de licencia revisada
Se configurará un nodo make con webhook para indicar que la licencia ha sido revisada y comenzar el workflow de Make que gestiona la documentación con la clínica.

Parámetos de entrada:

    + Id de la licencia creada

No hay parámetros de salida
