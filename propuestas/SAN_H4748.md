## Estructura

### Evento de creación de trato
Se programará un endpoint en Eneboo para crear un nuevo trato.

Parámetros de entada:
    + CIF del cliente
    + Nombre del contacto
    + Email del contacto

Acciones:

    + Crear contacto, si no existe, con:
        + Nombre del contacto
        + Email del contacto

    + Crear trato con:
        + Cliente: Cliente de Eneboo buscado por CIF
        + Nombre del trato: Nombre del cliente en Eneboo
        + Valor del trato: 0€
        + Contacto. Contacto creado / obtenido de BD
        + Tipo: Licencia Aloinjerto.
        + Familia de producto: ninguna.
        + Fecha y hora: las correspondientes al momento de creación.
        + Agente: Agente asociado al cliente
        + Agente supervisor: Agente supervisor parametrizado (al principio fijo para Paloma, para que ella revise)

Parámetros de salida:
    + Id del trato creado, que se debe almacenar en la excel de tratos para usarse como parámetro en las siguientes llamadas.

### Evento de envío de acuerdos para su firma

Parámetros de entada:
    + Id del trato
    + Tipo de evento: Fijo > ENVIO_ACUERDOS

Acciones:

    + Crear una tarea con:
        + Trato, el trato indicado
        + Descripción "Envío de acuerdos para su firma"
        + Estado: Terminada
        + Fecha / hora: Las de la llamada.

Parámetros de salida:
    (ninguno)

### Evento de recepción de acuerdos firmados
Mismo proceso que _Evento de envío de acuerdos para su firma_ con:

    + Tipo de Evento: Fijo > RECEPCION_ACUERDOS
    + Descripción de tarea: "Recepción de acuerdos firmados"

### Evento de envío de documentos para presentar
Mismo proceso que _Evento de envío de acuerdos para su firma_ con:

    + Tipo de Evento: Fijo > ENVIO_DOCS
    + Descripción de tarea: "Envío de documentos para presentar"

### Trato ganado, creación de licencias
Cuando el trato se gana, el usuario lo pone como Ganado.

~~Cuando un trato de tipo Licencia Aloinjerto se da como ganado, si no existe ya una licencia para este trato, se mostrará el formulario de alta de licencia, preguntando al usuario:~~
    ~~+ Fecha de inicio: `DUDA ¿usar algún valor por defecto, como la fecha de alta del trato?`~~
    ~~+ Fecha de caducidad.~~

Cuando un trato de tipo Licencia Aloinjerto se da como ganado, si no existe ya una licencia para este trato, se creará un registro de licencia, con los datos de fecha de inicio y de caducidad vacíos.

Otros datos de la licencia que se obtendrán directamente del trato son:
    + Cliente.
    + Tipo de licencia: Fijo > Tratamiento de regeneración ósea
    + Id. Trato asociado.

El usuario introduce estos datos y se crea una licencia asociada al trato.
