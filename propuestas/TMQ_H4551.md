# #TMQ #H4551 Cambios tras reunión 17/11/25

Los objetivos de este proyecto son:
+ Informar al cliente de los descuentos aplicados en los presupuestos / proformas
+ Modificar la nomenclatura de códigos de ejercicio, albarán y factura para establecer el año actual y distinguir albaranes de facturas
+ Unificar el formato de pedidos, albaranes y facturas igualándolo al de presupuesto / proforma
+ Incluir un texto o marca de agua en facturas para indicar que la factura está asegurada

### Presupuestos y proformas / Descuento en líneas
Incluiremos un campo calculado que se dibujará bajo el campo importe (= precio unitario x cantidad) de las líneas del informe impreso. El texto del campo será:

+ Si la línea no tiene descuentos, texto en blanco
+ Si la línea tiene descuento lineal, "Descuento de XX.XXX,XX€", donde X es la resta entre precio total y precio unitario x cantidad
+ Si la linea tiene descuento porcentual y no lineal, "Descuento del XX,XX%"

### Nomenclatura de ejercicios y facturas
Se implementará una nueva forma de generar los códigos de factura.

Nota: Se asume que desde 2026, los códigos de ejercicio seguirán el patrón "TMAA", donde AA es el año del ejercicio. Ejemplos: TM26,TM27, etc.

De esta forma, los códigos de presupuestos, pedidos, albaranes cumplirán el formato:

TMAASSNNNNNN donde:

+ TMAA es el código de ejercicio
+ SS es el código de serie
+ NNNNNN es el número de factura

Ejemplos TM260A000001, TM26IE000001, etc.

Para el caso de facturas el formato será:

FAASS/NNNNNN donde:

+ AA son los dos últimos caracteres del código de ejercicio. Para 'TM26' será '26'.
+ SS y NNNNNN son el código de serie y el número de factura

Ejemplos F260A/000001, F26IE/000001, etc.

Este cálculo del código de factura solo se realizará si el código de ejercicio comienza por 'TM', de forma que se active automáticamente con los documentos del ejercicio de 2026.

Recomendamos usar el código completo (FAASS/NNNNNN) tanto en la impresión de las facturas como en el envío a Hacienda a través de Verifactu.
