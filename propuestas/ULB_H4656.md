# #ULB #H4656 Control de rentabilidad y margen en presupuestos

Los objetivos de este proyecto son:

+ Conocer la rentabilidad y el margen de cada presupuesto.
+ Tener un punto de control para decidir si los presupuestos se fabrican o no en función de su rentabilidad.

## Estructura

### Facturación / Presupuestos / Campos
Añadiremos a la tabla de presupuestos los siguientes campos

+ Margen
+ Beneficio

El margen se calculará como el importe del presupuesto (Sin IVA) menos las comisiones menos los costes de los productos.

El coste de cada producto es su precio teórico (articulos.precioteorico)

El beneficio se calculará como margen / precio de venta (Sin IVA)

### Facturación / Presupuestos / Formulario y cálculo
El formulario de presupuestos tendrá una pestaña Rentabilidad que mostrará:

+ Margen
+ Beneficio
+ Botón explicativo del margen
+ Botón explicativo del beneficio
+ Botón de recalcular datos

El botón explicativo del margen mostrará la siguiente información:

```
Venta: [Total sin IVA del presupuesto]
Costes: [Total de costes]
    + Comisiones: [Total sin IVA del presupuesto x comisión del agente]
    + Productos: [Total de costes de productos]
        + Producto 1: [Precio teórico] x cantidad = [Precio teórico x cantidad]
        + ...
        + Producto n: [Precio teórico] x cantidad = [Precio teórico x cantidad]

Margen: [Total sin IVA del presupuesto] - [Total de costes] = [margen]
```

El botón explicativo del beneficio mostrará la siguiente información:
```
Venta: [Total sin IVA del presupuesto]
Costes: [Total de costes]
Margen: [Total sin IVA del presupuesto] - [Total de costes] = [margen]

Beneficio: [Margen] / [Total sin IVA del presupuesto] = [Beneficio]
```

El botón de recálculo calculará los valores de margen y beneficio en base a los costes actuales y actualizará los dos campos.

### Facturación / Configuración / Umbral de beneficio
Crearemos un nuevo campo en el formulario de configuración para indicar el umbral:

+ Umbral de beneficio. Decimal

### Facturación / Presupuestos / Cambio de estado
Cuando un presupuesto en estado _PENDIENTE_ cambia de estado (botón Cambiar estado):

+ Se calcula su margen y beneficio
    + Si el beneficio supera el umbral, pasa a estado _EN ESTUDIO_ (comportamiento actual)
    + Si el beneficio no supera el umbral, pasa a un nuevo estado _EN REVISION EC._ (En revisión económica)

Cuando un presupuesto en estado _EN REVISION EC._ cambia de estado (botón Cambiar estado), el presupuesto pasa a EN ESTUDIO (mismo mensaje que cuando pasa de _PENDIENTE_ a _EN ESTUDIO_ superando el umbral).

Es decir, creamos un nuevo estado _EN REVISION EC._ por el que pasarán los presupuestos que no superen el umbral de beneficio.


### Facturación / Presupuestos / Control de acceso
Solo el grupo de usuarios _admin_ podrá ver los datos de margen y beneficio, tanto en la tabla como en el formulario.
