# #VLC #H4576 Adaptaciones logísticas

Los objetivos de este proyecto son:
+ Poder realizar un seguimiento de las plantas que se venden desde que se compran
+ Poder realizar un seguimiento de los recultivos realizados
+ Conocer e incluir en el pasaporte fitosanitario la finca de origen de la planta vendida.

## Estructura

### Almacén / Extensión de artículos por lotes
Incluiremos la extensión de artículos por lotes, que permitirá crear lotes y movimientos de lotes para las plantas y productos que lo necesiten.

### Almacén / Nuevo campo de lote proveedor y cálculo de código interno
El campo de lote de los lotes comprados se calculará como:
```
+ VV es el litraje (obtenido de contenedor_planta.codcontenedor)
RRRRR-VV-LLLLL
```
Donde:

+ RRRRR es la referencia del artículo
+ LLLLL es un código de lote secuencial (000001, 000002, etc.)

Crearemos un nuevo campo en lotes:
+ Código proveedor: Que permitirá indicar el código del lote del proveedor de compra.

### Finca y pasaporte fitosanitario
En el apartado C de las etiquetas del pasaporte fitosanitario el código de lote se completará de la siguiente forma:
```
RRRRR-VV-LLLLL-FFF

P0150-P0150-00000001-CAÑA
P0150-P0150-00000002-f2
```
Donde:

+ FFF es el código de finca (almacén) desde el que se envía la mercancía.

### Lotes en pedidos pendientes
Modificaremos el formulario de líneas de pedido de cliente para poder asociar lotes disponibles de fincas al pedido.

El formulario mostrará una tabla con todos los lotes disponibles, con las siguientes columnas:
+ Almacén
+ Finca
+ Código de lote
+ Cantidad disponible

Mediante un botón podremos seleccionar un lote de esta tabla y asociarlo a la línea. Teniendo en cuenta que:

+ Si el lote tiene más cantidad disponible que la línea, se reserva toda la cantidad pendiente de la línea.

+ Si el lote tiene menos cantidad disponible que la línea, se reserva toda la cantidad disponible del lote, quedando pendiente cierta cantidad en la línea que debe reservarse a otro lote.

Estas reservas generarán movimientos de stock pendientes asociados al pedido y los lotes.

### Envío de pedidos a fincas e informe para fincas
Se modificará el proceso de pedido a fincas para obtener los datos de la tabla de movimientos de stock.

Se modificará el informe de preparación que se imprime en las fincas para indicar los códigos de lote a preparar.

### Albaranado de pedidos
Se modificará el proceso de albaranado para no preguntar los lotes enviados, sino usar los lotes y fincas indicados en el pedido a albaranar.

### Informe de etiquetas de pedido
Se modificará el informe de etiquetas de pedido para usar la tabla de movimientos de stock asociados a cada línea de pedido.

### Recultivos
Los recultivos se reflejarán en el programa mediante la opción de _Trazabilidad Interna_, indicando movimientos positivos para la planta crecida (mayor litraje) y creando nuevos lotes para ella, y movimientos negativos para la planta origen (menor litraje).

### Transferencias
Será necesario indicar las transferencias de lotes de plantas entre almacenes ¿fincas?. Para ello usaremos la opción de transferencias del menú de almacén.

Revisaremos la opción para asegurarnos de que los lotes se pueden transferir sin problemas.
