# #VLC #H4678 Integración con API de Crédito y Caución para control de riesgo

Los objetivos de este proyecto son:

+ Conocer el riesgo máximo autorizado de cada cliente
+ Establecer puntos de control que avisen al usuario cuando un cliente rebasa su riesgo máximo

## Estructura

### Clientes / Cálculo de Riesgo 
Modificaremos el cálculo del riesgo actual de cada cliente para calcularlo como suma de:

+ Recibos pendientes
+ Albaranes por facturar
+ Pedidos sin albaranar

### Clientes / Carga de riesgo máximo
Crearemos un proceso diario que llamará a la API de Crédito y Caución y obtendrá los riesgos máximos de los clientes contratados, actualizando el campo _Riesgo Máximo_ de sus fichas.

Crearemos un campo en la configuración de tesorería que indicará la última fecha de carga correcta de datos de C & C.

### Presupuestos / Comprobación de riesgo
Se establecen dos puntos de control en los presupuestos:

+ Si al guardar un presupuesto, el riesgo actual del cliente más el importe del presupuesto (IVA incluido) supera el riesgo máximo.
+ Si al aprobar un presupuesto, el riesgo actual del cliente más el importe del presupuesto (IVA incluido) supera el riesgo máximo.

Si el cliente tiene un riesgo máximo establecido (un valor no vacío en el campo), se mostrará el aviso:
```
El riesgo actual del cliente incluyendo este presupuesto es de X€, supera su riesgo máximo (Y€)
```

### Pedidos / Comprobación de riesgo
Se establecen dos puntos de control en los presupuestos:

+ Si al guardar un pedido, el riesgo actual del cliente incluyendo el nuevo importe del pedido supera el riesgo máximo. Tendremos en cuenta los casos de:
    + Alta (riesgo + importe)
    + Cambio (riesgo actual +- diferencia)

Si el cliente tiene un riesgo máximo establecido (un valor no vacío en el campo), se mostrará el aviso:
```
El riesgo actual del cliente incluyendo este pedido es de X€, supera su riesgo máximo (Y€)
```
